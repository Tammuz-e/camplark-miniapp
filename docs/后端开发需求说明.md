# 找嘢营 后端及管理端开发需求说明

## 一、我已理解的前端逻辑

### 1. 数据模型
| 实体 | 说明 | 关键字段 |
|------|------|----------|
| **营地 Camp** | 露营地点 | id, campName, location, latitude, longitude, terrainType, price, rating, imgs, desc |
| **营位 Spot** | 营地下的可预订单位 | id, campName, spotName, area, priceNum, priceUnit, facilities, desc, 关联 campId |
| **体验 Experience** | 营地/俱乐部提供的活动 | id, campName, campSiteName, activityType, price, rating, 关联 campId 或 clubId |
| **俱乐部 Club** | 用户可加入的社群 | id, clubName, tags, price, members, desc |
| **订单 Order** | 营位预订 | id, orderNo, spotId, checkIn, checkOut, nights, guests, totalPrice, status(待入营/已入营/已完成) |
| **评价 Review** | 营位评价 | id, orderId, spotId, userName, rating, content, date |
| **用户 User** | 小程序用户 | 微信登录，nickName, avatarUrl，收藏、加入俱乐部存在本地 |

### 2. 业务流程
- **预订流程**：选营位 → 填入住/退房、人数 → 填预定人/电话 → 提交订单（目前仅本地 mock）
- **评价流程**：订单完成后可评价
- **收藏/加入俱乐部**：目前存本地 Storage
- **成立俱乐部**：点击后提示「请联系客服」

---

## 二、计划实现的两个管理端

### 1. 平台管理端
- **职能**：平台运营方，管理全部数据
- **功能**：营地/营位/体验/俱乐部审核与管理、订单汇总、用户管理、数据统计、平台设置、俱乐部入驻审核

### 2. 俱乐部/营地管理端
- **职能**：营地或俱乐部运营方，只管理自己的数据
- **功能**：
  - **营地**：管理本营地的营位、体验、订单
  - **俱乐部**：管理本俱乐部、活动、成员
